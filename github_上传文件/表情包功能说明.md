# 表情包功能说明

机器人现在会在聊天时自己挑表情包发，群聊和私聊都支持。

## 怎么用

1. 每个角色有自己的表情包文件夹：`memes/<角色名>/`（如 `memes/玛修/`），互不共用
2. 文件名就是表情含义，例如 `无语.jpg`、`狗头.gif`、`震惊.png`（jpg/png/gif/webp 都行，别带 `[ ]` 符号）
3. 重启机器人后生效；之后再加图、删图**不用重启**

## 工作原理

- 每次回复时，程序把该角色文件夹里所有表情名告诉 AI
- AI 觉得气氛合适就在回复里写 `[表情:无语]`
- 程序把这个标记换成图片、带 `subType=1` 标记发给 NapCat——在 QQ 里以**表情样式**显示（小图贴纸，显示为"动画表情"），不是大图照片
- 已限制：一次回复最多发 1 个表情，避免刷屏
- 说明：QQ 原生 face/商城表情只能用腾讯内置的，自定义图片做不成那种；subType=1 的表情图片是最接近的形式

## 可选配置（角色卡 json 里加，不加就用默认值）

```json
"memes": {
  "enabled": true,
  "dir": "memes/玛修",
  "max_per_reply": 1,
  "offer_prob": 0.25,
  "min_interval": 180
}
```

- `enabled`: 设为 false 关闭该角色的表情包
- `dir`: 自定义文件夹路径，不填默认 `memes/<角色名>`（几个角色想共用一套图时可都指向同一个文件夹）
- `max_per_reply`: 一次回复最多发几个表情
- `offer_prob`: 每次回复把表情列表提供给 AI 的概率（默认 0.25——没给列表 AI 就不会发，这是控制频率的主闸门）
- `min_interval`: 两次发表情的最小间隔秒数（默认 180）；就算 AI 写了表情标记，不到间隔也会被强制拦下

## 频率控制原理（双重闸门）

以前每次回复都把表情列表塞给 AI，只靠提示词里一句"偶尔发"约束，AI 基本管不住自己。
现在：四分之三的回复 AI 根本看不到表情列表（闸门①），看到了想发也得满足 180 秒间隔（闸门②）。
想更克制就调低 `offer_prob`、调高 `min_interval`。

## 去白底脚本

AI 生成/网页下载的 PNG 常带白底，用 `去白底.py` 批量转透明：

```
python 去白底.py                  # 处理 memes/玛修，输出到 memes/玛修/透明
python 去白底.py memes/某角色      # 处理别的文件夹
```

只去掉连通图片边缘的白色背景，角色身上的白色不会被挖穿；顺带把大图缩到 512px（QQ 表情够用、发送快）。
**注意：机器人只读 `memes/<角色名>/` 这一层**，检查「透明」文件夹效果满意后，把里面的图移到上一级替换原图。

## 小技巧

- 文件名用情绪词效果最好：嘲笑、害羞、生气、装傻、问号、流汗……
- 想让某个角色更爱发表情，在它的 persona 里加一句"你很喜欢发表情包"
- 控制台日志若出现 `[表情] 找不到表情包「xx」`，说明 AI 编了个不存在的名字，已自动跳过，不影响发文字
