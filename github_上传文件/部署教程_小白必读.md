# 拟人 QQ 群机器人 · 小白部署教程

> 一句话理解整套系统：
> **NapCat** 让一个 QQ 号登录上线、收发消息 →
> **bot.py**（你的程序）拿到群消息，判断该不该回、回什么 →
> **DeepSeek**（AI）按你写的「角色卡」生成像真人的回复 →
> 再经 NapCat 发回群里。
>
> 想养第 2、第 3 个机器人？复制一张角色卡、再开一个 NapCat 就行，程序不用改。

---

## ⚠️ 先看三条重要提醒

1. **必须用小号**。QQ 机器人走的是非官方协议，腾讯有概率封号。请用一个不重要的 QQ 号，别用主号、别绑重要资料。
2. **新号先养几天**。刚注册的号容易被风控。建议号养几天、加好友、进群、正常聊几句再挂机器人。
3. **别刷屏**。回复太频繁、太快、整齐划一最容易被举报/风控。本程序已内置「随机概率 + 冷却时间 + 打字延迟」来降低风险，但你自己也别把概率调太高。

## 一、需要准备什么

| 东西 | 说明 | 花费 |
|------|------|------|
| 一个云服务器 | Linux 系统，让机器人 24 小时在线 | 国内轻量服务器约 30–60 元/月 |
| 一个小号 QQ | 给机器人用 | 免费 |
| 一个 DeepSeek API Key | 让 AI 生成回复 | 按量付费，闲聊用量极低，几块钱能用很久 |
| 本套代码 | 我给你的 qqbot 文件夹 | 免费 |

---

## 二、第一步：买云服务器（约 10 分钟）

国内常见选择：阿里云 / 腾讯云的「轻量应用服务器」，学生或新人有优惠。

1. 任选一家，买**轻量应用服务器**，系统镜像选 **Ubuntu 22.04** 或 **Debian 12**。
2. 配置 **2 核 2G** 就够（跑一个机器人）。想跑很多个就选 2核4G 以上。
3. 买好后记下服务器的**公网 IP**、**用户名(一般是 root 或 ubuntu)**、**密码**。
4. 用 **SSH 工具**连上去。Windows 推荐用 **MobaXterm** 或系统自带的 `ssh` 命令；Mac 直接用「终端」输入：
   ```
   ssh root@你的服务器IP
   ```
   输入密码即可进入。

> 进去后你会看到一个黑色命令行界面，后面所有命令都在这里输入。

---

## 三、第二步：装 NapCat（让 QQ 上线）

NapCat 是目前最稳的 QQ 协议端，官方提供一键脚本。在服务器命令行里执行：

```bash
curl -o napcat.sh https://nclatest.znin.net/NapNeko/NapCat-Installer/main/script/install.sh && sudo bash napcat.sh
```

> 这是 NapCat 官方安装脚本。如果链接失效，去官方文档找最新命令：https://napneko.github.io

装好后按提示启动，然后**登录你的机器人小号 QQ**：
- 一般会让你用**手机 QQ 扫码**登录，或在网页面板里扫码。
- NapCat 自带一个 **WebUI 网页面板**（默认地址类似 `http://你的服务器IP:6099`），扫码、设置都在这里完成。
- 第一次打开 WebUI 需要 token，安装时命令行会显示，照着输入即可。

### 关键：在 NapCat 里开启 WebSocket 服务

进入 NapCat 的 WebUI → 「网络配置」→ 新建一个 **WebSocket 服务端（正向 WS / WebSocket Server）**：
- **端口**填 `3001`（和角色卡里的 `ws_url` 对应）
- 设一个 **access_token**（一串你自己定的密码，比如 `mysecret123`），记下来
- 保存并启用

这样我们的 bot.py 就能通过 `ws://127.0.0.1:3001` 连上它。

---

## 四、第三步：拿 DeepSeek API Key

1. 打开 https://platform.deepseek.com ，注册/登录。
2. 充值一点点钱（充 10 元能聊很久）。
3. 在「API Keys」页面**新建一个 Key**，复制保存好（形如 `sk-xxxxxxxx`）。
   > Key 只显示一次，记得马上存下来。

---

## 五、第四步：把代码放上服务器并配置

1. 把我给你的 **qqbot 文件夹**整个上传到服务器（用 MobaXterm 左侧拖拽，或用 `scp` 命令）。假设放在 `/root/qqbot`。

2. 装 Python 依赖：
   ```bash
   cd /root/qqbot
   apt update && apt install -y python3-pip
   pip3 install -r requirements.txt
   ```

3. 改 **config.json**（填 DeepSeek Key）：
   ```bash
   nano config.json
   ```
   把 `deepseek_api_key` 的值换成你的 `sk-xxxx`。改完按 `Ctrl+O` 回车保存，`Ctrl+X` 退出。

4. 改 **角色卡 characters/傲娇少女_示例.json** 里这几项：
   - `self_id`：填机器人**小号的 QQ 号**
   - `napcat.access_token`：填你在 NapCat 里设的那个 token（要一致）
   - `enabled_groups`：填你想让它活动的**群号**，比如 `[123456789]`。留 `[]` 表示所有群都活动。

---

## 六、第五步：启动机器人

```bash
cd /root/qqbot
python3 bot.py characters/傲娇少女_示例.json
```

看到 `已连上 NapCat，开始监听群消息……` 就成功了！

去群里 **@它** 或喊它名字「小凛」，它就会回复。平时它也会偶尔随机插话。

### 让它后台长期运行（关掉 SSH 也不停）

用 `screen`：
```bash
apt install -y screen
screen -S linbot                       # 新建一个后台窗口
python3 bot.py characters/傲娇少女_示例.json
# 按 Ctrl+A 再按 D 退出窗口（程序继续在后台跑）
# 想回来看：screen -r linbot
```

## 七、怎么做新角色卡 / 养更多机器人

### 改人设（最常调的地方）
打开角色卡，重点改 **`persona`** 这段——它就是给 AI 的"性格说明书"，写得越具体，越像那个角色。例子见示例卡里的傲娇少女。

其它可调参数：

| 字段 | 作用 | 建议 |
|------|------|------|
| `name` | 机器人名字 | 角色名 |
| `nicknames` | 群友喊哪些名字会必回 | 多写几个外号 |
| `keywords` | 出现这些词必回 | 如 "求助"、"晚安" |
| `reply_probability` | 平时随机插话概率 | 0.05 左右；想活跃点调 0.1，太高会刷屏 |
| `engagement.boost_probability` | 插话后"追聊"概率 | 0.7，越高越爱连续聊 |
| `engagement.window_seconds` | 追聊持续时间(秒) | 120 |
| `engagement.max_followups` | 一轮最多追聊几次 | 4 |
| `cooldown_seconds` | 两次主动发言最小间隔 | 8，防刷屏 |
| `context_size` | AI 能看到多少条历史 | 16，越大越懂上下文但越费钱 |
| `typing_delay` | 打字延迟，模拟真人手速 | 默认即可 |

> 注意：被 @、叫名字、关键词 这三种**一定会回**，不受概率和冷却影响。
> 随机插话和追聊才受 `reply_probability` / `cooldown` 控制。

### 养第二个机器人
1. 准备**另一个小号 QQ**。
2. 复制示例角色卡 → 改名比如 `characters/元气少女_小桃.json`，改里面的 `name`、`persona`、`self_id`。
3. 在服务器上**再开一个 NapCat 实例**登录第二个号，WebSocket 端口换成别的（比如 `3002`），并把这个端口填进新角色卡的 `ws_url`。
4. 再开一个 screen 窗口跑：
   ```bash
   python3 bot.py characters/元气少女_小桃.json
   ```

每个机器人互相独立，互不影响。

## 八、常见问题

**Q：连不上 NapCat / 一直在重连？**
检查：① NapCat 的 WebSocket 服务是否已启用、端口是否是 3001；② 角色卡里的 `access_token` 和 NapCat 里设的是否一模一样；③ QQ 是否已扫码登录在线。

**Q：@它没反应？**
① 看运行 bot.py 的窗口有没有打印"触发"日志。没有就是消息没收到（多半是 NapCat 没连上）；② 有"DeepSeek 调用失败"就是 API Key 或余额问题；③ 确认 `self_id` 填的是机器人自己的 QQ 号。

**Q：回复太频繁/太啰嗦？**
调小 `reply_probability`，调大 `cooldown_seconds`；想让回复更短，改 config.json 的 `max_tokens` 小一点（如 120）。

**Q：deepseek-chat 模型以后会不会失效？**
DeepSeek 偶尔会更新模型名。如果哪天报模型错误，去 config.json 把 `deepseek_model` 换成 DeepSeek 文档里当前的最新模型名即可，其它都不用动。

**Q：被封号了？**
换个小号重新登录 NapCat 即可，代码和角色卡都不用动。平时把概率调低、别让它发太敏感的内容，能降低风险。
